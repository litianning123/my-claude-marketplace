# Loop Creator

[English](README.md)

一个 Claude Code 插件，通过引导式对话帮你设计和生成循环（loop）配置。

## 功能

- 通过对话了解你的自动化目标（什么需要重复？如何验证？何时停止？）
- 推荐合适的输出格式（命令、项目文件夹或可复用技能）
- 生成现成的 loop 配置，包含安全边界、验证清单和调度说明

## 安装

将 `loop-creator/` 目录复制到你的 Claude Code 插件目录中。

## 快速开始

使用以下短语触发：
- "为我的每日站会创建一个 loop"
- "设置一个 CI 监控 loop"
- "我想自动化我的每周代码审查"

向导会引导你完成就绪检查、范围定义和调度 — 然后生成你需要的文件。

## 分步使用

### 1. 启动向导

说出上述触发短语之一。向导即开始阶段 1 — 就绪关卡。

### 2. 回答 11 个问题

向导在三个阶段中逐个提问：

**阶段 1：就绪关卡（Q1-Q6）** — 尽早过滤掉"不适合作为首个 loop"的场景。
- Q1：这个 loop 应该执行什么任务？
- Q2：应该多久运行一次？
- Q3：你将如何验证输出？
- Q4：Claude 需要访问哪些资源？
- Q5：什么情况下应该停止或升级？
- Q6：如果出错，最坏的情况是什么？

如果你的回答过于模糊，向导会追问并帮助你缩小范围。

**阶段 2：范围与形态（Q7-Q9）** — 定义每次迭代做什么。
- Q7：逐步描述一次迭代的过程
- Q8：每次运行是否需要记住之前运行的结果？
- Q9：这是为你个人使用，还是你的团队也需要复用？

**阶段 3：节奏与自主性（Q10-Q11）** — 确定调度和权限。
- Q10：应该从什么权限级别开始？
- Q11：固定时间表还是自主调节？

### 3. 审查层级推荐

向导根据你的回答打分并推荐输出层级：

> 基于你的答案：得分 4 — 多步骤工作流、有状态、需要人工审核关卡、验证复杂。推荐：一个包含 TASK.md、LOOP_INSTRUCTIONS.md 和 PROGRESS.md 的项目文件夹。可以吗？

确认或选择不同的层级。

### 4. 审查生成的输出

向导生成你的 loop 配置。对于项目和技能层级，会在写入文件之前询问保存位置。

### 5. 先手动运行（必须）

每个输出都包含"在调度之前先手动运行"的说明。手动运行 loop 3-5 次，检查输出，然后在调度之前完善指令。

### 6. 调度

手动运行稳定后，使用输出中包含的 `/loop` 命令进行调度。

## 输出层级

| 层级 | 产出内容 | 适合场景 |
|------|----------|----------|
| Command | 可直接粘贴的 `/loop` 命令 | 简单轮询、无状态检查 |
| Project | 完整的 loop 工作区（TASK.md、LOOP_INSTRUCTIONS.md、PROGRESS.md、outputs/） | 有状态、多步骤工作流 |
| Skill | 可复用的 SKILL.md + 参考实现 | 团队工作流、重复出现的模式 |

各层级的具体示例见 `references/example-outputs.md`。
