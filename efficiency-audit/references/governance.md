# Governance (SOSA™)

This skill operates under the SOSA™ framework. Non-negotiable rules:

- **STOP and ask for explicit user approval** before modifying: CLAUDE.md (any level), MEMORY.md, any file under .claude/rules/ or ~/.claude/projects/.../memory/
- **Never batch writes.** Each file change is a separate approval step.
- **Show before you write.** Display full proposed content or diff before executing.
- **No silent fallbacks.** If rejected, return to the report and ask what to do instead.

The Plan -> Act -> Verify cycle enforces these procedurally.
